## 2. Auditoria de falhas sistêmicas (PR #6)

### 🔴 Críticas

1. **Conflito de hash de senha (login quebrava).**
   Coexistiam dois esquemas: cadastro salvava em **SHA-256** (64 chars) e
   `migrar_senhas.py` gerava **bcrypt** (60 chars). O `verificar_senha` decidia
   pelo tamanho `== 64`, então hashes bcrypt caíam no ramo "texto puro" e o
   **login falhava sempre** para esses usuários.
   - Os dados reais confirmaram o problema: **9 perfis bcrypt** + **11 SHA-256**
     na tabela `aniversariantes`.
   - **Correção:** `hash_senha` agora gera bcrypt; `verificar_senha` aceita
     bcrypt + SHA-256 + texto puro (retrocompatível). Nenhuma senha foi
     corrompida e ninguém precisou redefinir.

2. **`bcrypt` ausente em `requirements.txt`** (era importado por
   `migrar_senhas.py`). Adicionado.

3. **`migrar_senhas.py` corromperia hashes SHA-256** (re-hasheava como se
   fossem texto puro). Agora preserva bcrypt **e** SHA-256; só migra texto puro.

### 🟠 Importantes

4. **Prefixo de página `3_` duplicado** (`3_Colaboradores.py` +
   `3_📊_Pesquisa.py`) → ordem ambígua no menu. Renomeado para
   `4_📊_Pesquisa.py`.

5. **Nome usado como chave única.** A edição de perfil passou a usar o `id` do
   registro (`.eq("id", ...)`), removendo o `.single()` por nome (que quebrava
   com homônimos) e avisando quando há nomes duplicados.

### 🟡 Menores

6. **`hash()` não determinístico** na rotação dos post-its (`app.py`) — variava
   por execução (`PYTHONHASHSEED`). Trocado por `hashlib` determinístico.

7. **`README.md` ausente** (referenciado pelo devcontainer). Criado.

## 3. Reset de senha de perfil (PR #7)

As senhas são hash (irreversíveis) e a tabela não tem coluna de e-mail, então
não havia recuperação. Foi adicionado um fluxo de reset operado pelo admin.

### Pelo painel admin (recomendado)

1. Entre no modo admin (barra lateral).
2. Abra o expander **"🔓 Resetar senha de perfil"**.
3. Selecione a pessoa (`nome (#id)`) e clique em **Resetar senha**.

O registro recebe `perfil_completo = false` e `senha_perfil = NULL`. A pessoa
então cria uma senha nova na aba **Cadastro**. Foto, curiosidade e data são
preservados.

### Pelo SQL do Supabase (alternativa)

Por `id` (mais seguro):

```sql
UPDATE public.aniversariantes
SET perfil_completo = false,
    senha_perfil   = NULL
WHERE id = 9;   -- troque pelo id da pessoa
```

Por nome:

```sql
UPDATE public.aniversariantes
SET perfil_completo = false,
    senha_perfil   = NULL
WHERE nome = 'Natallia Pereira Magio';
```

Conferir o id antes:

```sql
SELECT id, nome, perfil_completo FROM public.aniversariantes ORDER BY nome;
```
